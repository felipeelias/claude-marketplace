# Blog Post Format

Apply when writing for the Jekyll blog.

## Structure Templates

### Tutorial

1. Context/problem (1-2 sentences, max 1 paragraph)
2. State what was built or what the solution is
3. Walk through the steps (labeled "Step 0, 1, 2...")
4. End when the content is done. No summary, no "wrapping up", no call to action.

### Announcement / Show-and-tell

1. What's new (1 sentence)
2. Screenshot or code immediately
3. Feature list or walkthrough
4. Done.

### Short-form Note

Just the thought. No framing, no conclusion.

## Signature Moves

- **"Tired of X, built Y"**: quick personal motivation, one sentence, then move on.
- **Show before explain**: screenshot or code within the first few lines.
- **Fair alternatives**: mention other tools/approaches, link to them.
- **Credit people**: link to those who inspired the work.
- **Abrupt endings**: when the content is done, stop. No wrap-up paragraph.
- **Practical closing notes**: if anything, end on a troubleshooting tip or a single useful detail.

## Markdown Conventions

- Reference-style links (`[text][id]`), definitions at the bottom, sorted alphabetically.
- Link products/tools on first mention only.
- Backticks for CLI tools/commands (`jq`, `claude-notifier`), plain text for product names (ntfy, Tailscale).
- `##` for top-level sections, `###` for subsections.
- Title comes from front matter, not a heading in the body.
